# Frontend Rules

## Rule 1 — Component Reusability
All UI elements must be modular and reusable.

### Reason
Improves scalability and maintainability.

---

## Rule 2 — Mobile Responsiveness
The website must function correctly on all screen sizes.

### Requirements
- Responsive grids
- Adaptive typography
- Mobile navigation
- Optimized touch interactions

---

## Rule 3 — Performance Optimization
Heavy horror effects must not reduce performance.

### Requirements
- Lazy load assets
- Compress images
- Optimize animations
- Avoid unnecessary re-renders

---

## Rule 4 — Accessibility Compliance
The website must remain readable and accessible.

### Requirements
- Semantic HTML
- Proper contrast ratios
- Keyboard accessibility
- Accessible forms

---

## Rule 5 — Clean Code Standards
Maintain organized frontend architecture.

### Requirements
- Feature-based folder structure
- Reusable utilities
- Consistent naming
- Comment important logic
